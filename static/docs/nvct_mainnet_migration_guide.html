NVCT Mainnet Migration Guide

Update (May 2025): NVC Banking Platform is migrating NVCT from Ethereum Sepolia testnet to Ethereum mainnet. Follow this guide to ensure a smooth transition.

1. Migration Overview

The NVC Token (NVCT) is transitioning from Ethereum Sepolia testnet to Ethereum mainnet, which represents a significant milestone in the platform's development. This migration allows:

  • Real-world value transfer using NVCT on the public Ethereum network
  • Integration with major decentralized exchanges and DeFi platforms
  • Enhanced token security through Ethereum's robust mainnet infrastructure
  • Expanded interoperability with other blockchain systems

2. Technical Changes

The migration involves deploying three smart contracts to Ethereum mainnet:

Contract Description Function
NVCToken ERC-20 token implementation Core NVCT token functionality
SettlementContract Payment settlement processor Handles transfer settlements between parties
MultiSigWallet Multi-signature authorization Secures high-value transactions with multiple approvals

3. Migration Timeline

  • Phase 1: Contract Deployment (Current)
    Smart contracts being deployed to Ethereum mainnet
  • Phase 2: Dual Operation (1-2 weeks)
    Both testnet and mainnet supported while systems are validated
  • Phase 3: Full Mainnet Transition (After validation)
    Complete transition to mainnet-only operation

4. What You Need To Do

For platform administrators and operators:

  1. Environment Configuration: Set the following environment variables: 
    # Ethereum mainnet settings
ETHEREUM_NETWORK=mainnet
INFURA_PROJECT_ID=your_infura_project_id
ADMIN_ETH_PRIVATE_KEY=your_private_key  # Only for contract deployment
  2. Contract Deployment: Use the provided migration tool to deploy contracts: 
    # Deploy each contract individually
python mainnet_migration.py deploy --contract=settlement_contract
python mainnet_migration.py deploy --contract=multisig_wallet
python mainnet_migration.py deploy --contract=nvc_token

# Verify deployment
python mainnet_migration.py validate
  3. System Verification: Test system functionality after migration:
    • Verify token transfer operations
    • Test payment settlement workflows
    • Confirm multi-signature transaction processes

For end users and API integrators:

  • No immediate action is required
  • Updated API documentation will be provided after the migration is complete
  • Web interfaces will automatically use the mainnet implementation

5. Security Considerations

Important: Ethereum mainnet transactions require ETH for gas fees. Ensure your deployment wallet has sufficient ETH balance before proceeding with migration.

Additional security measures for mainnet operation:

  • Keep private keys secure and never expose them in code or public repositories
  • Follow multi-signature approval workflows for high-value transactions
  • Monitor gas prices to optimize transaction costs
  • Implement robust error handling for mainnet transactions
  • Regularly audit smart contract interactions for security vulnerabilities
